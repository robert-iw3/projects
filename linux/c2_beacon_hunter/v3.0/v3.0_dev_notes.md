**v3.0 Program Increment 3 – Status Report**

**Epic 1: Promiscuous eBPF Parser (Wire-Speed Ingestion)**

**Date:** 2026-03-07

**Status:** Development completed, awaiting testing.

### Overview
Epic 1 is fully developed and ready for testing:

The system now supports two modes via `config.ini`:
- `mode = host` → Original kprobe/tracepoint behaviour (unchanged)
- `mode = promisc` → New raw packet parser on any interface (SPAN/mirror or promiscuous NIC)

### Completed Stories

**Story 1.1: TC/XDP program in promiscuous mode**
- Full XDP parser (`c2_promisc.bpf.c`) with robust bounds checking
- IPv4 + IPv6 support (5-tuple extraction)
- In-kernel ringbuf emission for flows
- Safe attach/detach logic (prevents -EBUSY)

**Story 1.2: Robust Ethernet/IP/TCP/UDP header parsing**
- Complete header parsing with early-exit safety
- Payload sampling (first 64 bytes) for future entropy
- Zero-copy, wire-speed operation

**Story 1.3: In-kernel flow state tracking (5-tuple maps)**
- Ringbuf-based flow events (no per-packet spam)
- Dual-mode loader selection (`c2_promisc_loader` vs original)

### Files Updated

- `c2_promisc.bpf.c` – New XDP parser (IPv4 + IPv6)
- `c2_promisc_loader.c` – Robust native loader with clean shutdown
- `Makefile` – Standalone build for promiscuous binaries
- `libbpf_collector.py` – Dual-mode support + IPv6 + loopback filter
- `collector_factory.py` – Mode-aware factory (host/promisc)
- `c2_beacon_hunter.py` – Mode logging + full backward compatibility
- `docker-compose.yaml` – Mounts v3.0/, passes `MODE` env var
- `health_check.sh` – Reports current mode, loader type, XDP status

### Key Features Added
- Configurable via `[general] mode = promisc` (default = host)
- Full IPv6 future-proofing in kernel
- Paramount error handling (bounds checks, graceful fallback, safe detach)
- Health check & Docker awareness of new mode
- Zero performance impact in host mode
