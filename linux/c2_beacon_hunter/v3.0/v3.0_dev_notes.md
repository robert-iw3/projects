**Date:** 2026-03-07

**Status:** Development completed, awaiting testing.

### Overview

- The ML engine has been evolved from process-level to **network-level (subnet/CIDR)** behaviour.
- Baselines and clustering now operate on CIDR blocks (/24 for IPv4, /64 for IPv6) instead of individual processes.
- This dramatically improves detection of subnet-wide C2 campaigns while reducing alert fatigue on single-host noise.

### Completed Stories

**Story 1.1: TC/XDP program in promiscuous mode**
- Full XDP parser (`c2_promisc.bpf.c`) with robust bounds checking
- IPv4 + IPv6 support (5-tuple extraction)
- In-kernel ringbuf emission for flows
- Safe attach/detach logic (prevents -EBUSY)

**Story 1.2: Robust Ethernet/IP/TCP/UDP header parsing**
- Complete header parsing with early-exit safety
- Payload sampling (first 64 bytes) for future entropy
- Zero-copy, wire-speed operation

**Story 1.3: In-kernel flow state tracking (5-tuple maps)**
- Ringbuf-based flow events (no per-packet spam)
- Dual-mode loader selection (`c2_promisc_loader` vs original)

**Story 2.1:**
- BPF Hash Maps for interval, entropy, CV, and packet-size tracking

**Story 2.2:**
- Kernel-side aggregation before ringbuf submission

→ Dramatically reduces ringbuf pressure and user-space CPU load. Events are now aggregated every 8 packets (or on new flow).

**Story 3.1:** CIDR-based baselines (10.0.5.0/24, etc.) — implemented in `baseline_learner.py`

**Story 3.2:** 3D+ clustering with flow metadata (interval + entropy + subnet diversity score) — implemented in `BeaconML.py`

### Key Features Added
- Configurable via `[general] mode = promisc` (default = host)
- Full IPv6 future-proofing in kernel
- In-kernel flow state tracking (5-tuple + interval/CV/packet stats)
- Kernel-side aggregation (reduces ringbuf events by ~87%)
- Paramount error handling (bounds checks, graceful fallback, safe detach)
- Health check & Docker awareness of new mode
- Zero performance impact when running in host mode
